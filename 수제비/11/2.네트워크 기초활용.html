원하는 정보를 원하는 수신자 또는 기기에 정확하게 전송하기 위한 기반 인프라

OSI 7계층
1. 사용자와 네트워크 간 응용서비스 연결, 데이터 생성
2. 데이터 형식 설정,부호교환, 암복호화
3. 송수신 간의 논리적인 연결
4. 송수신 프로세스 간의 연결
5. 단말기 간 데이터 전송을 위한 최적화된 경로 제공
6. 인접 시스템 간 데이터 전송,전송 오류 제어
7. 0과 1의 비트 정보를 회선에 보내기 위한 전기적 신호 변환

1계층 장비
1. 여러 대의 컴퓨터를 연결하여 네트워크로 보내거나 하나의 네트워크로 수신된 정보를 여러 대의 컴퓨터로 송신하기 위한 장비
2. 디지털 신호를 증폭시켜 주는 역할을 하여 신호가 약해지지 않고 컴퓨터로 수신되도록 하는 장비

2계층 장비 1. 두 개의 근거리 통신망을 서로 연결해 주는 통신망 연결 장치
2. 느린 전송 속도의 브리지, 허브의 단점을 개선하기 위해서, 출발지에서 들어온 프레임을 목적지 MAC주소 기반으로 빠르게 전송시키는 데이터 링크 계층의 통신장비
3. 외부 네트워크와 접속하여 가장 빠른 속도로 데이터를 주고받은 수 있게 컴퓨터내에 설치되는 장치
4. 스위치 기능을 가진 허브

3계층 장비
1. LAN과 LAN 혹은 LAN과 WAN을 연겨하기 위한 인터넷 네트워킹 장비
2. 프로토콜을 서로 다른 통신망에 접속할 수 있게 해주는 장비
3. 3계층에서 네트워크 단위들을 연결하는 통신장비
4. 외부로부터 들어오는 인터넷 라인을 연결하여 유선으로 여러 대의 기계를 연결하거나 무선 신호로 송출하면서 여러 대의 컴퓨터가 하나의 인터넷 라인을 공유할 수 있도록 하는 네트워크 장비
5. 광역 네트워크를 커버하는 스위칭 허브

4계층 장비
4계층에서 네트워크 단위들을 연결하는 통신 장비

서로 다른 시스템이나 기기들 간의 데이터 교환을 원활히 하기 위한 표준화된 통신규약

프로토콜 3요소
1. 시스템 간의 정보 전송을 위한 데이터 형식, 코딩, 신호 레벨들의 규정
2. 시스템 간의 정보 전송을 위한 제어 정보로 조정과 에러 처리를 위한 규정
3. 시스템 간의 정보 전송을 위한 속도 조절과 순서 관리 규정

컴퓨터나 원거리 통신 장비 사이에서 메시지를 주고받는 양식과 규칙의 체계

프로토콜 특징
1. 전송이 가능한 작은 블록으로 나누어지는 기법
2. 단편화되어 온 조각들을 원래 데이터로 복원하는 기법
3. 상위 계층의 데이터에 각종 정보를 추가하여 하위 계층으로 보내는 기법
4. 데이터의 전송량이나 속도를 제어하는 기법
5. 전송 중 잃어버리는 데이터나 오류가 발생한 데이터를 검증하는 제어 기법
6. 송신과 수신 측의 시점을 맞추는 기법
7. 하나의 통신 회선에 여러 기기들이 접속할 수 있는 기술
8. 송신과 수신지의 주소를 부여하여 정확한 데이터 전송을 보장하는 기법

링크의 설정과 유지 및 종료를 담당하며 노드 간의 회선 제어, 흐름제어, 오류제어 기능을 수행하는 계층

데이터링크 계층 프로토콜 종류
1. 점대점 방식이나 다중방식의 통신에 사용되는 ISO에서 표준화한 동기식 비트 중심의 데이터 링크 프로토콜
2. 네트워크 분야에서 두 통신 노드 간의 직접적인 연결을 위해 일반적으로 사용되는 데이터 링크 프로토콜
3. 프로토콜 처리를 간략화하여 단순히 데이터 프레임들의 중계기능과 다중화 기능만 수행함으로써 데이터 처리속도의 향상 및 전송지연을 감소시킨 고속의 데이터 전송 기술
4. 정보전달의 기본단위를 53바이트 셀 단위로 전달하는 비동기식 시분할 다중화 방식의 패킷형 전송 기술

다양한 길이의 패킷을 네트워크들을 통해 전달하고, 그 과정에서 전송 계층이 요구하는 서비스 품질을 위한 수단을 제공하는 계층

네트워크 계층 프로토콜 종류
1. 송수신 간의 패킷 단위로 데이터를 교환하는 네트워크에서 정보를 주고받는 데 사용하는 통신 프로토콜
2. IP네트워크 상에서 IP주소를 MAC주소로 변환하는 프로토콜
3. IP호스트가 자신의 물리 네트워크 주소는 알지만 IP주소를 모르는 경우, 서버로부터 IP주소를 요청하기 위해 사용하는 프로토콜
4. IP패킷을 처리할 때 발생되는 문제를 알려주는 프로토콜
5. 호스트 컴퓨터와 인접 라우터가 멀티캐스트 그룹 멤버십을 구성하는 데 사용하는 통신 프로토콜
6. 데이터 전송을 위해 목적지까지 갈 수 있는 여러 경로 중 최적의 경로를 설정해주는 라우터 간의 상호 통신 프로토콜

인터넷에서 사용되는 패킷 교환 네트워크상에서 데이터를 교환하기 위한 32비트 주소체계를 갖는 네트워크 계층의 프로토콜

IPv4가 가지고 있는 주소고갈,보안성,이동성 지원 등의 문제점을 해결하기 위해 개발된 128bit주소체계를 갖는 차세대 인터넷 프로토콜

IPv6의 특징
1. 32->128bit 주소공간 제공
2. 네트워크의 물리적 위치에 제한받지 않고 같은 주소를 유지하면서도 자유롭게 이동가능
3. 패킷 출저 인증과 데이터 무결성 및 비밀 보장 기능을 ip프로토콜 체계에 반영
4. 흐름 레이블 개념을 도입, 특정 트래픽은 별도의 특별한 처리를 통해 높은 품질의 서비스 제공
5. ipv6 호스트는 ipv6네트워크에 접속하는 순간 자동적으로 네트워크 주소를 부여받음
6. 자동 네트워킹 및 인터넷 연결 지원
7. ip패킷의 처리를 신속하게 할 수 있도록 고정 크기의 단순 헤더를 사용하는 동시에 확장 헤더를 통해 가능에 대한 확장 및 옵션 기능의 사용이 용이한 구조
8. 흐름 레이블을 사용하여 패킷의 흐름을 실시간 제공

ipv4 vs ipv6의 전송방식 차이

IPv4에서 IPv6로 전환방법
1. 두가지 프로토콜이 모두 탑재되어 있고 통신 상대방에 따라 해당 ip스택을 선택하는 방법
2. ipv6망에 인접한 ipv4망을 거쳐 다른 ipv6망으로 통신할 때 ipv4망에 터널을 만들고 ipv4에서 사용하는 프로토콜로 캡슐화하여 전송하는 방법
3. 주소변환 기술은 ipv4망과 ipv6망 사이에 주소변환기를 사용하여 서로 다른 네트워크상의 패킷을 변환시키는 방법

캐스트 프로토콜 종류
1. 인터넷에서 같은 내용의 데이터를 여러 명의 특정한 그룹의 수신자들에게 동시에 전송할 수 있는 프로토콜
2. 고유 주소로 식별된 하나의 네트워크 목적지에 1:1로 트래픽 또는 메시지를 전송하는 프로토콜
3. 하나의 송신자가 같은 서브 네트워크상의 모든 수신자에게 데이터를 전송하는 프로토콜
4. 단일 송신자로부터의 데이터그램을 토폴로지상의 잠재적인 수신자 그룹안에서 가장 가꾸은 노드로 연결시키는 전송 프로토콜

데이터 전송을 위해 목적지까지 갈 수 있는 여러 경로 중 최적의 경로를 설정해주는 라우터 간의 상호 통신규약이다.

AS내에서 사용하는 거리 벡터알고리즘에 기초하여 개발된 내부 라우팅 프로토콜

RIP특징
1. 거리 벡터 라우팅 기반 메트릭 정보를 인접 라우터와 주기적으로 교환하여 라우팅 테이블을 갱신
2. 최대 홉 수 15개로 제한
3. UDP포트 번호 520
4. 30초마다 전체 라우팅 정보를 브로드캐스팅

규모가 크고 복잡한 TCP/IP 네트워크에서 RIP의 단점을 개선하기 위해 자신을 기준으로 링크 상태 알고리즘을 적용하여 최단 경로를 찾는 라우팅 프로토콜

OSPE특징
1. 링크 상태 라우팅 기반 메트릭 정보를 한 지역내 모든 라우터에 변경이 발생했을 때만 보내고 라우팅 테이블을 구성,계산
2. 최소 지연,최대 처리량 등 관리자가 라우팅 메트릭 지정
3. 자치 시스템을 지역으로 나누어 라우팅을 효과적으로 관리
4. 홉 카운트에 제한없음

AS 상호간에 경로 정보를 교환하기 위한 라우팅 프로토콜

라우팅 알고리즘 종류
1. 인접 라우터와 정보를 공유하여 목적지까지의 거리와 방향을 결정하는 라우팅 프로토콜 알고리즘
2. 링크 상태 정보를 모든 라우터에 전달하여 최단 경로 트리를 구성하는 라우팅 프로토콜 알고리즘

전송 제어 프로토콜이며 프로그램 간에 일련의 옥텟을 안정적으로 순서대로 에러없이 교환할 수 있게 해주는 프로토콜

TCP 특징
1. 패킷 손실,중복,순서 바뀜 등이 없도록 보장
2. 같은 전송계층의 UDP가 비연결성인 것과는 달리 TCP는 연결 지향적임
3. 흐름 제어 기능을 활용하여 송신 및 수신속도를 일치시킴
4. 네트워크가 혼잡하다고 판단될 때는 혼잡제어 기법을 사용하여 송신율을 감속함

비연결성이고 신뢰성이 없으며 순서화되지 않은 데이터그램 서비스를 제공하는 전송계층의 통신 프로토콜

UDP의 특징
1. 데이터그램 지향의 전송계층용 프로토콜임
2. 수신된 메시지의 순서를 맞추지 않음
3. 빠른 요청과 응답이 필요한 실시간 응용에 적합
4. 헤더는 고정 크기의 8바이트만 사용

응용 프로그램 간의 대화를 유지하기 위한 구조를 제공하고 이를 처리하기 위해 프로세스들의 논리적인 연결을 담당하는 계층

세션 계층 프로토콜 2가지
1. 원격 프로시저 호출이라고 불리며 원격 제어를 위한 코딩없이 다른 주소 공간에서 함수나 프로시저를 실행할 수 있는 프로세스 간 통신에 사용되는 프로토콜
2. 응용계층의 애플리케이션 프로그램에게 AIP를 제공하여 상호 통신할 수 있도록 해주는 프로토콜

애플리케이션이 다루는 정보를 통신에 알맞은 형태로 만들거나 하위계층에서 온 데이터를 사용자가 이해할 수 있는 형태로 만드는 역할을 담당하는 계층

표현 계층의 프로토콜
1. 이미지를 위해 만들어진 표준 규격
2. 멀티미디어를 위해 만들어진 표준 규격

응용 프로세스와 직접 관계하여 일반적인 응용 서비스를 수행하는 역할을 담당하는 계층

응용 계층의 프로토콜 종류
1. 텍스트 기반의 통신규약으로 인터넷에서 데이터를 주고받을 수 있는 프로토콜
2. TCP/IP 프로토콜을 가지고 서버와 클라이언트 사이의 파일을 전송하기 위한 프로토콜
3. 인터넷에서 TCP 포트번호 25번을 사용하여 이메일을 보내기 위해 이용되는 프로토콜
4. 응용 계층 인터넷 프로토콜중 하나로, 원격 서버로부터 TCP/IP연결을 통해 이메일을 가져오는 데 사용하는 프로토콜
5. 원격 서버로부터 TCP/IP 연결을 통해 이메일을 가져오는데 사용하는 프로토콜
6. 인터넷이나 로컬 영역에서 네트워크 연결에 사용되는 네트워크 프로토콜

컴퓨터 네트워크와 통신의 방식 중 하나로 작은 블록의 패킷으로 데이터를 전송하며 데이터를 전송하는 동안만 네트워크 자원을 사용하도록 하는 통신 방식

통신을 원하는 두 단말장치가 패킷 교환망을 통해 패킷을 원활히 전달하기 위한 통신 프로토콜

네트워크 리소스를 특정 사용층이 독점하도록 하는 통신 방식
