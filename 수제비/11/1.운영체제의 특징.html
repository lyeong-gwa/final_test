사용자가 컴퓨터의 하드웨어를 쉽게 사용할 수 있도록 인터페이스를 제공해 주는 소프트웨어

운영체제 특징
1. 한정된 시스템 자원을 효과적으로 사용할 수 있도록 관리 및 운영
2. 컴퓨터 시스템과 사용자를 연결함
3. 다중 사용자와 다중 응용 프로그램 환경하에서 자원의 현재 상태를 파악하고 자원 분배를 위한 스케줄링을 담당
4. CPU, 메모리 공간, 기억장치,입출력 장치등의 자원을 관리함
5. 입출력 장치와 사용자 프로그램을 제어

운영체제의 핵심이 되는 기능들이 모여 있는 컴퓨터 프로그램

커널의 기능
1. 프로세스 스케줄링 및동기화 관리 담당
2. 프로세스에게 메모리 할당 및 회수 관리 담당
3. 입 출력 장치 스케줄링 및 전반적인 관리 담당
4. 관리 파일의 생성과 삭제, 변경,유지 등의 관리담당

운영체제 종류 1. 멀티태스킹 기능과 GUI환경을 제공하는 마이크로소프트사가 개발한 운영체제
2. 범용 다중 사용자 방식의 시분할 운영체제
3. 유닉스 기반으로 개발되고 소스 코드가 공개된 오픈 소스 기반의 운영체제
4. 애플이 유닉스 기반으로 개발한 그래픽 사용자 인터페이스 기반 운영체제
5. 휴대 전화를 비롯한 휴대용 장치를 위한 운영체제와 미들웨어, 사용자 인터페이스 그리고 표준 응용 프로그램을 포함하고 있는 운영체제

윈도즈 운영체제 특징
1. 키보드 없이 마우스로 아이콘이나 메뉴를 선택하여 작업을 수행하는 그래픽 기반의 인터페이스 방식
2. 동시에 여러 개의 프로그램을 실행하면서 운영체제가 각 작업의 CPU 이용 시간을 제어
3. 하드웨어를 설치했을 때 필요한 시스템 환경을 운영체제가 자동으로 구성해주는 자동감지 기능 제공
4. 개체를 현재 작성 중인문서에 자유롭게 연결 또는 삽입하여 편집할 수 있게 하는 기능 제공

유닉스 계열 운영체제 특징
1. 프롬프트가 나타난 상태에서 사용자가 명령을 입력하면 시스템은 그 명령을 수행하는 사용자 명령 기반의 대화식 운영체제 기능을 제공
2. 다수의 작업이 중앙처리장치와 같은 공용 자원을 나누어 사용하여 한 번에 하나 이상의 작업을 수행하는 기능 제공
3. 여러 대의 단말이 하나의 컴퓨터에 연결되어 여러 사람이 동시에 시스템을 사용하여 각각의 작업을 수행할 수 있는 기능 제공
4. 90%이상 C언어로 구현되어 있고 시스템 프로그램이 모듈화되어 있어 다른 하드웨어 기종으로 쉽게 이식 가능
5. 유닉스는 계층적 트리 구조로 통합적인 파일 관리가 용이

운영체제 핵심 기능
1. 프로그램의 실행이 종료될 때까지 메모리를 가용한 상태로 유지 및 관리하는 기능
2. 일시 중지 및 재실행, 동기화, 통신, 교착상태 처리, 프로세스 생성 삭제 등의 관리

메모리 관리 기법
1. 주기억장치에 적재할 다음 프로세스의 반입 시기를 결정하는 기법
2. 디스크에 있는 프로세스를 주기억장치의 어느 위치에 저장할 것인지 결정하는 기법
3. 실행해야 할 프로세스를 주기억장치에 어떤 방법으로 할당할 것인지 결정하는 기법
4. 재배치 기법으로 주기억장치에 있는 프로세스 중 어떤 프로세스를제거할 것인지를 결정하는 기법

메모리 배치 기법
1. 프로세스가 적재될 수 있는 가용 공간 중에서 첫 번째 분할에 할당하는 방식
2. 가용 공간 중에서 가장 크기가 비슷한 공간을 선택하여 프로세스를 적재하는 방식
3. 프로세스의 가용 공간들 중에서 가장 큰 공간에할당하는 방식

프로세스 상태
1. 사용자에 의해 프로세스가 생성된 상태
2. CPU를 할당받을 수 있는 상태
3. 프로세스가 CPU를 할당받아 동작 중인 상태
4. 프로세스 실행 중 입출력 처리 등으로 인해 CPU를 양도하고 입출력 처리가 완료까지 대기 리스트에서 기다리는 상태
5. 프로세스가 CPU를 할당받아 주어진 시간 내에 완전히 수행을 종료한 상태

프로세스 상태 전이
1. 준비 상태에 있는 여러 프로세스 중 실행될 프로세스를 선정하여 CPU에 할당
2. CPU를 할당 받은 프로세스는 지정된 시간이초과되면 스케줄러에 의해 PCB저장, CPU반납 후 다시 준비 상태로 전이
3. 실행 상태에 있는 프로세스가 지정된 할당시간을 초과하기 전에 입출력이나 기타 사건이 발생하면 CPU를 스스로 반납하고 입출력이 완료될 때까지 대기 상태로 전이됨
4. 어느 순간에 입출력이 종료되면 대기 상태의 프로세스에게 입출력 종료 사실을 wait&signal 등에 의해 알려주고 준비 상태로 전이됨

CPU를 사용하려고 하는 프로세스들 사이의 우선순위를 관리하는 작업

프로세스 스케줄링 주요 용어
1. 프로세스가 결과를 산출하기까지 소요되는 시간
2. 프로세스들이 입력되어 수행하고 결과를 산출하기까지 소요되는 시간
3. 대기 큐의 프로세스가 결과를 산출하기 소요되는 시간 평균
4. 프로세스가 프로세서에 할당 대기까지 큐에 대기하는 시간
5. 프로세스가 대기 큐에서 대기하는 평균 시간
6. 요구되는 포르세싱 시산을 모두 수행하고 종료된 시간
7. 한 프로세스가 프로세서를 독점하는 것을 방지하기 위해 서비스되는 시간 할당량
8. (대기시간+서비스 시간)/서비스 시간

프로세스 스케줄링 유형 2가지
1. 하나의 프로세스가 CPU를 차지하고 있을 때 우선순위가 높은 다른 프로세스가 현재 프로세스를 중단시키고 CPU를 점유하는 스케줄링 방식
2. 한 프로세스가 CPU를 할당 받으면 작업 종료 후 CPU반환 시까지 다른 프로세스는 CPU점유가 불가능한 스케줄링 방식

선점형 스케줄링 알고리즘 유형 4가지
1. 프로세스는 같은 크기의 CPU시간을 할당, 프로세스가 할당된 시간 내에 처리 완료 못하면 준비 큐 리스트의 가장 뒤로 보내지고 CPU는 대기중인 다음 프소세스로 넘어감
2. 가장 짧은 시간이 소요되는 프로세스를 먼저 수행, 남은 처리 시간이 더 짧다고 판단되는 프로세스가 준비 큐에 생기면 언제라도 프로세스가 선점된다
3. 작업들을 여러 종류 그룹으로 분할 여러 개의 큐를 이용하여 상위단계 작업에 의한 하위단계 작업이 선점 당함
4. 입출력 위주와 CPU위주인 프로세스의 특성에 따라 큐마다 서로 다른 CPU 시간 할당량을 부여함

비선점 스케줄링 알고리즘의 유형
1. 프로세스별로 우선순위가 주어지고 우선순위에 따라 CPU할당
2. 작업들이 명시된 시간이나 기한 내에 완료되도록 계획
3. 프로세스가 대기 큐에 도착한 순서에 따라 CPU를 할당함
4. 프로세스가 도착하는 시점에 따라 그 당시 가장 작은 서비스 시간을 갖는 프로세스가 종료 시까지 자원 점유
5. 대기 중인 프로세스 중 현재 응답률이 가장 높은 것을 선택

물리적인 리소스들을 사용자에게 하나로 보이게 하거나 하나의 물리적인 리소스를 여러 개로 보이게 하는 기술

가상화의 종류
1. 하드웨어 플랫폼 위에서 실행되는 호스트 프로그램이 게스트 프로그램을 만들어 마치 독립된 환경을 만들어 낸 것처럼 보여주는 기법
2. 게스트 소프트웨어 위에서 사용자는 독립된 하드웨어에서 소프트웨어가 실행되는 것처럼 활용하는 기법

가상화 종류
1. 물리적으로 컴퓨터 리소스를 가상화하여 논리적 단위로 리소스를 활용할 수 있도록 하는 기술
2. 스토리지와 서버 사이에 소프트웨어/하드웨어 계층을 추가하여 스토리지를 논리적으로 제어 및 활용할 수 있도록 하는 기술
3. 서버와 I/O디바이스 사이에 위치하는 미들웨어 계층으로 서버의 I/O자원을 물리적으로 분리하고 케이블과 스위치 구성을 단순화하여 효율적인 연결을 지원하는 기술
4. 컨테이너화된 애플리케이션들이 단일 운영체제상에서 실행되도록 해주는 기술
5. 여러 대의 컴퓨터계싼 및 저장능력을 이용하여 커다란 계산문제나 대용량의 데이터를 처리하고 저장하는 기술
6. 물리적으로 떨어져 있는 다양한 장비들을 연결하기 위한 수단으로 중계장치의 가상화를 통한 가상 네트워크를 지원하는 기술

인터넷을 통해 가상화된 컴퓨터 시스템 리소스를 제공하고 정보를 자신의 컴퓨터가 아닌 클라우드에 연결된 다른 컴퓨터로 처리하는 기술

클라우드 컴퓨팅 분류
1. 기업 또는 조직 내부에서 보유하고 있는 컴퓨팅 자원을 사용하여 내부에 구축되어 운영되는 클라우드
2. 클라우드 서비스 제공 업체에서 다중 사용자를 위한 컴퓨팅 자원 서비스를 제공하는 클라우드
3. 기업 또는 조직 내부 자원을 이용한 사설 클라우드와 공용 클라우드를 모두 사용하는 클라우드

클라우드 컴퓨팅 유형
1. 서버, 스토리지 같은 시스템 자원을 클라우드로 제공하는 서비스
2. 인프라를 생성,관리 하는 복잡함 없이 애플리케이션을 개발,실행,관리할 수 있게 하는 플랫폼을 제공하는 서비스
3. 소프트웨어 및 관련 데이터는 중앙에 호스팅되고 사용자는 웹 브라우저 등의 클라이언트를 통해 접속하여 소프트웨어를 서비스 형태로 이용하는 서비스
