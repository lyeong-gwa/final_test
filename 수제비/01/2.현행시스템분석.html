생성패턴
1.복잡한 인스턴스를 조립하여 만드는 구조 객체를 생성하는 방법과 객체를 구현하는 방법을 분리
2.처음부터 일반적인 원형을 만들어 놓고 필요한 부분만 수정하여 사용하는 패턴
3.상위 클래스에서 객체를 생성하는 인터페이스를 정의하고 하위 클래스에서 인스턴스를 생성하도록 하는 방식
4.구체적인 클래스에 의존하지 않고 서로 연관되거나 의존적인 객체들의 조합을 만드는 인터페이스를 제공하는 패턴
5.전역 변수를 사용하지 않고 객체를 하나만 생성하도록 하며 생성된 객체를 어디에서든지 참조할 수 있도록 하는 디자인 패턴





구조패턴
기능의 클래스 계층과 구현의 클래스 계층을 연결, 추상화된 부분과 실제 구현 부분을 독립적으로 확장
기존에 구현되어 있는 클래스에 필요한 기능을 추가해 나가는 설계 패턴
복잡한 시스템에 대해 단순한 인터페이스를 제공, 결합도를 낮춤
다수의 객체로 생성될 경우 모두가 갖는 본질적인 요소를 클래스화하여 공유함으로서 메모리 절약 , 클래스의 경량화
실체 객체에 대한 대리 객체, 미리 할당x상관없는 것을 실제 이용할 때 할당-> 메모리 절약 정보은닉 역할도하는 디자인패턴
객체들의 관계를 트리 구조로 구성
기존에 생성된 클래스를 재사용할 수 있도록 중간에서 맞춰주는 역할





행위패턴
중재자를 두고 중재자에게 요구, 통신 빈도수 줄이는 디자인패턴
언어의 다양한 해석, 구문의 해석을 맡는 클래스를 작성 문법의 캡슐화
컬렉션 구현 방법을 노출시키지 않고 모든 항목에 접근, 복잡 객체의 원소를 순차적으로 접근
작업의 일부분을 서브 클래스로 캡슐화
한 객체의 상태 바뀌먼 의존하는 다른 객체들에 연락
객체 상태를 캡슐화하여 클래스화 함으로 상태에 따라 다르게 처리
각 클래스 데이터 구조로부터 처리 기능을 분리 객체의 구조는 변경안하고 기능만 따로 추가, 확장
실행될 기능을 캡슐화하여 여러 기능을 실행할 수 있는 재사용성이 높은 클래스를 설계하는 패턴
알고리즘 군을 정의, 같은 알고리즘을 각각 하나의 클래스로 캡슐화한 다음 필요시 교환해서 사용
클래스 설계 관점에서 객체의 정보를 저장할 필요가 있을 때 적용하는 디자인패턴
정적으로 어떤 기능에 대한 처리의 연결이 하드코딩 되어 있을 때 사용하는 디자인패턴
