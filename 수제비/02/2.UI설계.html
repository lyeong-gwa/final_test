객체지향 소프트웨어 개발 과정에서 산출물을 명세화 시각화 문서화 할 때 사용되는 모델링 기술과 방법론을 통합해서 만든 표준화된 범용 모델링





UML의 특징
1. 개념 모델 작성시 오류가 적고 의사통이 용이
2. 다양한 프로그래밍 언어로 실행 시스템의 예측 가능
3. 정확한 모델 제시, 완전한 모델 작성 가능
4. 시스템에 대한 평가 및 의사소통의 문서





UML의 구성요소
1. 추상적인 개념으로 주체를 나타내는 요소
2. 사물의 의미를 확장하고 명확히 하는 요소
3. 사물과 관계를 모아 그림으로 표현한 형태





UML 다이어그램 구분 2가지





구조적 다이어그램 종류
1. 객체지향 모델링 시 클래스의 속성 및 연산과 클래스 간 정적인 관계를 표현한 다이어그램
2. 클래스에 속한 사물들, 인스턴스를 특정 시점의 객체와 객체 사이의 관계로 표현
3. 시스템을 구성하는 물리적인 컴포넌트와 그들 사이의 의존 관계를 나타내는 다이어그램
4. 컴포넌트 사이의 종속성을 표현하고 결과물 프로세스 컴포넌트 등 물리적 요소들의 위치를 표현하는 다이어그램
5. 복합체 구조 다이어그램은 클래스나 컴포넌트가 복합 구조를 갖는 경우 그 내부 구조를 표현하는 다이어그램
6. 유스케이스나 클래스 등의 모델 요소들을 그룹화한 패키지들의 관계를 표현한 다이어그램





행위적 다이어그램 종류
1. 시스템이 제공하고 있는 기능 및 그와 관련된 외부 요소를 사용자의 관점에서 표현
2. 객체 간 동적 상호작용을 시간적 개념을 중심으로 메세지 흐름으로 표현한 다이어그램
3. 동작에 참여하는 객체들이 주고 받는 메세지를 표현하고 메세지뿐만 아니라 연관까지 표현
4. 하나의 객체가 자신이 속한 클래스의 상태 변화 혹은 다른 객체와의 상호작용에 따라 상태가 어케 변화하는지 표현
5. 시스템이 어떤 기능을 수행하는지를 객체의 처리 로직이나 조건에 따른 처리의 흐름을 순서대로 표현
6. 객체 상태 변화와 시간 제약을 명시적으로 표현





UML의 기본적 요소 이외의 새로운 요소를 만들어내기 위한 확장 메커니즘





다이어그램 유형
사각형에 이름 속성 연산으로 이루어지고 관계가 이어져있다.





다이어그램 유형
졸라맨과 타원과의 관계가 그려져있다.





다이어그램 유형
사각형 객체 아래로 선과 긴 사각형이 뻗어있다.





다이어그램 유형
작은사각형과 큰사각형이 붙어있다.





원과 둥근모소리 사각형으로 구성되어있고 상태도처럼 생김





활동다이어그램처럼 생겼는데 사각형 내용이 상태를 표시한다.





졸라맨이 존재하며 메세지 주체가 사각형으로 메세지를 주고받는다.





사각형에 다른 작은 사각형들이 박혀있는 형태이다.





UML 스테레오 타입
1. 하나의 유스케이스가 어떤 시점에 반드시 다른 유스케이스를 실행하는 포함 관계
2. 하나의 유스케이스가 어떤 시점에 다른 유스케이스를 실행할 수도 아닐 수도 있는 확장관계
3. 모든 메서드가 추상 메서드이며 바로 인스턴스를 만들 수 없는 클래스로 추상 메서도와 상수만으로 구성된 클래스(사각형)
4. 일반적으로 정보 또는 오래 지속되는 연관 행위를 형상화하는 클래스 (으)
5. 시스템과 외부 액터와의 상호작용을 담당하는 클래스 (ㅏO)
6. 시스템이 제공하는 기능의 로직 및 제어를 담당 (회전판같음)





클래스 다이어그램 구성요소
1. 공통의 속성 연산 관계 의미를 공유하는 객체들의 집합
2. 클래스의 구조적 특성에 이름을 붙인 것
3. 객체에 요청하여 행동에 영향을 줄 수 있는 서비스
4. 클래스에 접근할 수 있는 정도를 표현





클래스 다이어그램 클래스간의 관계
1. 클래스가 서로 개념적으로 연결된 선 일반적인 선
2. 하나의 객체에 여러 개의 독립적인 객체들이 구성되는 관계 흰색 머리 콩나물 모양 선으로 연결
3. 영구적이고 집합 관계보다 더 강한 관계로 구성 끝이 검정색인 콩나물 모양 선으로 연결
4. 하나의 사물이 다른 사물에 비해 더 일반적인지 구체적인지 표현 머리빈 화살표
5. 하나의 클래스가 다른 크래스를 사용하는 관계 선으로 구성된 화살표
6. 추상 클래스나 인터페이스를 상속받아 자식클래스가 추상 메서드를 구현할 때 사용 extend implement 선 사용





객체 인스턴스를 생성하지 않고 단지 유사 클래스들의 공통된 특징을 정의, 하나 이상의 추상 메서드와 일반 필드 및 일반 메서드를 포함하는 클래스





기능을 모아놓은 클래스로 추상 메서드와 상수만을 포함하는 추상 클래스





유스케이스 다이어그램 구성요소
1. 시스템이 제공해야 하는 서비스, 기능 액터가 시스템을 통해 수행하는 일련의 행위
2. 사용자가 시스템에 대해 수행하는 역할
3. 전체 시스템의 영역을 표현
4. 발생되는 이벤트의 흐름
5. 사람, 시스템, 하드웨어, 시간의 흐름에 의해 시작





유스케이스 다이어그램 관계
1. 유스케이스를 수행할 때 다른 유스케이스가 반드시 수행되는 관계
2. 특정 조건에서 한 유스케이스로만 확장되는 관계
3. 추상적인 액터와 좀 더 구체적인 액터 사이에 맺어주는 관계





시퀸스 다이어그램 구성요소
1. 사각형 안에 밑줄 친 이름으로 명시
2. 객체로부터 뻗어 나가는 점선
3. 오퍼레이션이 실행되는 시간을 의미
4. 객체 간의 상호작용은 메세지 교환으로 이루어짐





UI 시나리오 문서의 작성 요건
1. 누락없고 빠짐없이 상세하게 기술
2. 사용자의 요구사항이 일관성이 있어야 함
3. 이해하기 쉽도록 구성 어려운 용어 사용X
4. 문서를 쉽게 읽을 수 있어야 한다
5. 쉽게 추적이 가능해야 하고 쉬워야 한다.
6. 쉽게 변경 가능해야 한다.





화면 설계 도구 종류
1. 파워포인트의 지원툴
2. 스케치한 느낌으로 빠르게
3. 카카오에서 제작한 온라인 프로토타이핑 도구





프로토타이핑 도구
1. 웹 브라우저 통해 와이어 프레임과 프로토타이핑 작업 동시에
2. 스토리보드에 포함되는 정책, 플로 차트, 디스크립션까지 모두 작성 가능한 도구
3. 네이버이서 만든 스토리보드 플로차트 등 기능 지원 도구





UI 디자인 도구
1. 다양한 목업 템플릿 활용 가능 레ㅣ아웃 중심의 UI 디자인 설계 지원도구
2. UI 디자인에 최적화된 툴로 쉽고 직관적인 인터페이스를 제공





UI 디자인 산출물로 작업하는 프로토타이핑 도구
1. 간단한 인터랙션 적용 가능
2. 모바일 앱 프로토타이핑 최적화
3. 커피 스크립트라고 하는 개발 언어 사용
