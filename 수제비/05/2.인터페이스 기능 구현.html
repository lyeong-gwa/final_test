인터페이스 구현 방법 두가지

데이터 통신을 사용하는 인터페이스 구현 방법
1. 인터페이스 객체를 생성하기 위해서 데이터베이스에 있는 정보를 SQL을 통하여 선택한 후 이를 제이슨으로 생성
2. 송신측에서 제이슨으로 작성된 인터페이스 각체를 AJAX 기술로 수신측에 송신하면 수신측은 인터페이스 객체를 수신받고 파싱 후 처리 이후 송신측으로 TRUE/FALSE전달

인터페이스 개체를 사용하는 인터페이스 구현
1. 송신 관련 정보를 관리하기 위한 항목과 송신 시스템에서 필요한 항목 구현
2. 수신 관련 정보를 관리하기 위한 항목과 수신 시스템에서 필요한 항목 구현

인터페이스 개발시 보안 취약점을 방지하기 위한 가이드

시큐어 코딩 가이드 적용대상
1. 프로그램 입력값에 대한 검증 누락, 부적절한 검증 ,잘못된 형식 지정
2. 보안 기능의 부적절한 구현
3. 거의 동시에 수행 지원하는 병렬 시스템, 하나 이상의 프로세스가 동작하는 환경에서 시간 및 상태의 부적절한 관리
4. 에러 미처리 등
5. 개발자가 범할 수 있는 오류
6. 기능성이 불충분한 캡슐화로 인해 인가되지 않은 사용자에게 데이터 누출
7. 의도된 사용에 반하는 방법으로 API를 사용하거나 보안에 취약한 API의 사용

데이터베이스 암호화 알고리즘 종류 3가지
1. 암호화 알고리즘의 한 종류로 암 복호화에 같은 암호 키를 쓰는 알고리즘
2. 공개키는 누구나 알 수 있지만 그에 대응하는 비밀키는 키의 소유자만이 알 수 있도록 공개키와 비밀키를 사용하는 알고리즘
3. 해시값으로 원래 입력값을 찾아낼 수 없는 일방향성의 특성을 가진 알고리즘

데이터베이스 암호화 기법 4가지
1. 애플리케이션 레벨에서 암호모듈을 적용하는 애플리케이션 수정 방식
2. 암 복호화 모듈이 DB 서버에 설치된 방식
3. DB서버의 DBMS 커널이 자체적으로 암복호화 기능을 수행하는 방식
4. API방식과 PLUG-IN방식을 결합하는 방식

인터페이스 데이터의 암호화 전송 보안 기술
1. IP 계층에서 무결성과 인증을 보장하는 인증 헤더와 기밀성을 보장하는 암호화를 이용하여 양 종단 간 구간에 보안 서비스를 제공하는 터널링 프로토콜
2. 전송계층과 응용계층 사이에서 클라이언트와 서버 간의 웹 데이터 암호화, 상호 인증 및 전송 시 데이터 무결성을 보장하는 보안 프로토콜
3. 웹상에서 네트워크 트래픽을 암호화하는 주요 방법 중 하나로서 클라이언트와 서버 간에 전송되는 모든 메세지를 각각 암호화하여 전송
