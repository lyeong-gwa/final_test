다수의 인원, 시스템 또는 프로그램이 사용할 목적으로 통합하여 관리되는 데이터의 집합
통합된, 저장된, 운영, 공용 데이터

데이터베이스 특성
1. 쿼리에 대하여 살시간 응답
2. 새로운 데이터에 대한 항상 최신의 데이터 유지
3. 다수의 사용자가 동시에 같은 내용의 데이터 이용
4. 사용자가 요구하는 데이터 내용으로 데이터를 찾음

1. 파일에 이름을 부여하고 저장 검색을 위해 논리적으로 어디에 위치할지 정의
2. 관계형 모델을 기반으로 하는 가장 보편화된 데이터베이스 관리시스템
3. 상하 종속적인 관계로 계층화하여 관리하는 데이터 모델
4. 네트워크상의 망상 형태로 표현한 데이터 모델

데이터관리의 복잡성을 해결하는 동시에 데이터 추가, 변경, 검색, 삭제 및 백업 복구 보안 등의 기능 지원 소프트웨어

DBMS유형
1. 유니크한 키에 하나의 값을 가지고 있는 형태
2. 키 안에 컬럼, 값 조합으로 여러 개의 필드를 갖는 DBMS
3. 값의 데이터 타입이 문서라는 타입을 사용하는 DBMS
4. 시맨틱 웹과 온톨로지 분야에서 활용되는 그래프로 데이터를 표현하는 DBMS

DBMS 특징
1. 부적절한 자료가 입력되어 동일한 내용에 대하여 서로 다른 데이터가 저장되는 것을 허용하지 않는다.
2. 삽입, 삭제,갱신, 생성 후에도 저장된 데이터가 변함없이 일정
3. 장애가 발생시 특정 상태로 복구될 수 있음
4. 불법적인 노출,변경, 손실로부터 보호되어야 하는 성질
5. 응답시간, 저장 공간 활용 등이 최적화되어 요구조건을 만족시켜야 하는 성질

시스템, 서비스, 조직등에서 주어진 비용, 시간 내에 처리 가능한 데이터 범위를 넘어서는 수십 페타바이트 크기의 비정형 데이터

빅데이터 특성
1. 페타바이트 수준의 대규모 데이터
2. 정형, 비정형, 반정형의 다양한 데이터
3. 빠르게 증가하고 수집되며 처리되는 데이터

데이터 저장에 고정된 테이블 스키마가 필요하지 않고 조인연산을 사용할 수 없으며, 수평적으로 확장이 가능한 DBMS

대규모로 저장된 데이터 안에서 체계적이고 자동적으로 통계적 규칙이나 패턴을 찾아내는 기술

데이터 마이닝 주요 기법
1. 과거 데이터로부터특성을 찾아내어 분류모형을 만들어 이를 토대로 새로운 레코드의 결과 값을 예측
2. 데이터 안에 존재하는 항목들 간의 종속 관계를 찾아내는 기법
3. 연관 규칙에 시간 관련 정보가 포함된 형태의 기법
4. 대상 레코드들을 유사한 특성을 지닌 몇 개의 소그룹으로 분할하는 작업으로 작업의 특성이 분류규칙과 유사

실세계에 존재하는 모든 개념과 개념들의 속성 그리고 개념 간의 관계 정보를 컴퓨터가 이해할 수 있도록 서술해 놓은 지식베이스

대량의 텍스트 데이터로부터 패턴 또는 관계를 추출하여 의미 있는 정보를 찾아내는 기법

웹으로부터 얻어지는 방대한 양의 정보로부터 유용한 정보를 찾아내기 위하여 분석하는 기법

NoSQL의 특성중 하나인 BASE의 설명
1. 언제든지 데이터는 접근할 수 있어야 한다.
2. 노드의 상태는 내부에 포함된 정보에 의해 결정되는 것이 아니라 외부에서 전송된 정보를 통해 결정
3. 일정 시간이 지나면 데이터의 일관성이 유지됨

NoSQL의 유형
1. 유니크한 키에 하나의 값을 가지고 있는 형태
2. 키안에 컬럼,값 조합으로 된 여러 개의 필드를 갖는 db
3. 값의 데이터 타입이 document라는 타입을 사용하는 db
4. 시맨틱 웹과 온톨로지 분야에서 활용되는 그래프로데이터를 표현하는 db
