데이터베이스 시스템에서 하나의 논리적 기능을 정상적으로 수행하기 위한 작업의 기본 단위

트랜잭션 특성 4가지
1. 분해가 불가능한 작업의 최소단위이며 하나라도 실패할 경우 전체가 취소되어야하는 특성
2. 트랜잭션이 실행 성공 후 항상 일관된 데이터베이스 상태를 보존해야하는 특성
3. 트랜잭션 실행 중 생성하는 연산의 중간 결과를 다른 트랜잭션이 접근 불가한 특성
4. 성공이 완료된 트랜잭션의 결과는 영속적으로 데이터베이스에 저장하는 특성

트랜잭션의 상태변화
1. 초기 상태 트랜잭션이 실행 중일 때 가지는 상태
2. 마지막 명령문이 실행된 후에 가지는 상태
3. 트랜잭션이 성공적으로 완료된 후 가지는 상태
4. 정상적인 실행이 더이상 진행될 수 없을 때 가지는 상태
5. 트랜잭션이 취소되고 데이터베이스가 트랜잭션 시작 전 상태로 환원된 상태

트랜잭션 제어 언어는 TCL이며 종류는 ~,~,~가 있다.
1. 트랜잭션을 메모리에 영구적으로 저장하는 명령어
2. 트랜잭션내역을 저장 무효화시키는 명령어
3. 롤백을 위한 시점을 저장하는 명령어

다수 사용자 환경에서 여러 트랜잭션을 수행할 때 데이터베이스 일관성 유지를 위해 상호작용을 제어하는 기법

병행 제어 미보장 시 문제점 4가지
1. 먼저 실행된 트랜잭션의 결과를 나중에 실행된 트랜잭션이 덮어쓸 때 발생하는 오류
2. 트랜잭션의 중간 수행 결과를 다른 트랜잭션이 참조하여 발생하는 오류
3. 두 트랜잭션이 동시에 실행되어 데이터베이스의 일관성이 결여되는 오류
4. 복수의 트랜잭션이 데이터 공유시 특정 트랜잭션이 처리를 취소할 경우 트랜잭션이 처리한 곳의 부분을취소하지 못하는 오류

병행 제어 기법의 종류
1. 같은 자원을 액세스하는 다중 트랜잭션 환경에서 db의 일관성과 무결성을 유지하기 위해 트랜잭션의 순차적 진행을 보장하는 직렬화 기법
2. 트랜잭션이 어떠한 검증도 수행하지 않고 일단 트랜잭션을 수행하고 트랜잭션 종료 시 검증을 수행하여 데이터베이스에 반영하는 기법
3. 트랜잭션과 트랜잭션이 읽거나 갱신한 데이터에 대해 트랜잭션이 실행을 시작하기 전에 타임 스탬프를 부여하여 부여된 시간에 따라 트랜잭션을 수행하는 기법
4. 트랜잭션의 타임스탬프와 접근하려는 데이터의 타임스탬프를 비교하여 직렬가능성이 보장되는 적절한 버전을 선택하여 접근하도록 하는 기법

다른 트랜잭션이 현재의 데이터에 대한 무결성을 해치지 않기 위해 잠금을 설정하는 정도

트랜잭션을 수행하는 도중 장애로 인해 손상된 데이터베이스를 손상되기 이전의 정상적인 상태로 복구시키는 작업

회복기법 종류
1. 트랜잭션이 완료되기 전까지 데이터베이스에 기록하지 않는 기법
2. 트랜잭션 수행 중 갱신 결과를 바로 데이터베이스에 반영하는 가법
3. 장애 발생 시 검사점 이후에 처리된 트랜잭션에 대해서만 장애 발생 이전의 상태로 복원시키는 회복 기법
4. 데이터베이스 트랜잭션 수행 시 복제본을 생성하여 데이터베이스 장애 시 이를 이용해 복구하는 기법

데이터를 정의하는 언어로 데이터를 담는 그릇을 정의하는 언어

DDL의 대상
1. 하나의 속성이 가질 수 있는 원자값들의 집합
2. 데이터베이스의 구조, 제약조건 등의 정보를 담고 있는 기본적인 구조
3. 데이터 저장 공간
4. 하나 이상의 물리 테이블에서 유도되는 가상의 테이블
5. 검색을 빠르게 하기 위한 데이터 구조

스키마 분류
1. 사용자나 개발자의 관점에서 필요로 하는 데이터베이스의 논리적 구조
2. 데이터의 전체적인 논리적 구조
3. 물리적 저장장치의 관점에서 보는 데이터베이스 구조

데이터를 저장하는 항목인 필드들로 구성된 데이터의 집합

테이블 용어
1. 테이블 내의 행을 의미
2. 태이블 내의 열을 의미
3. 여러 개의 집합체를 담고 있는 관계형 데이터베이스에서 각각의 구분할 수 있는 논리적 개념
4. 튜플의 개수
5. 애트리뷰트의 개수
6. 하나의 애트리뷰트가 취할 수 있는 같은 타입의 원자값들의 집합

데이터를 빠르게 찾을 수 있는 수단으로서 테이블에 대한 조회 속도를 높여 주는 자료 구조

인덱스 종류
1. 데이터가 정렬된 순서로 생성되는 인덱스
2. 해시 함수에 의해 직접 데이터에 키 값으로 접근하는 인덱스
3. 각 컬럼에 적은 개수 값이 저장된 경우 선택하는 인덱스
4. 수식이나 함수를 적용하여 만든 인덱스
5. 하나의 컬럼으로만 구성한 인덱스
6. 두개 이상의 컬럼으로 구성한 인덱스
7. 기본 키 기준으로 레코드를 묶어서 저장하는 인덱스

인덱스 스캔 방식
1. 인덱스 루트 블록에서 리프블록까지 수직적으로 탐색한 후에 리프 블록을 필요한 범위만 스캔하는 방식
2. 수직적 탐색 없이 인덱스 리프블록을 처음부터 끝까지 수평적으로 탐색하는 방식
3. 수직적 탐색만으로 데이터를 찾는 스캔 방식
4. 선두 컬럼이 조건 절에 빠졌어도 인덱스를 활용하는 스캔방식

DDL 명령어 종류
1. 생성
2. 변경
3.삭제
4.내용삭제

~는 데이터베이스에 저장된 자료들을 입력 수정 삭제 조회하는 언어

DML의 4가지 명령어
1. 조회
2. 삽입
3. 갱신
4. 삭제

두개 이상의 테이블을 연결하여 데이터를 검색하는 방법

데이터베이스 관리자가 데이터 보안 무결성 유지 병행 제어 회복을 위해 관리자가 사용하는 제어용 언어
