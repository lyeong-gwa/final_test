개발 도구의 분류
1. 작성한 코드의 빌드 및 배포를 수행하는 도구
2. 개발자의 코드 작성과 디버깅, 수정 등과 같은 작업을 지원하는 도구
3. 코드의 기능 검증과 전체의 품질을 높이기 위해 사용하는 도구
4. 개발자들이 작성한 코드와 리소스 등 산출물에 대한 버전 관리를 위한 도구

서버 하드웨어 개발환경
1. 웹 상의 정적 콘텐츠 처리, HTTP를 이용한 처리
2. 동적 콘텐츠를 처리
3. 데이터의 수집, 저장을 위한 용도
4. 파일 저장 용도

클라이언트 하드웨어 개발환경
1. 설치를 통해 사용자와 커뮤니케이션하는 프로그램
2. 웹 서비스의 형태로 서버에서 웹 애플리케이션을 응답시 브라우저를 통해 사용자와 커뮤니케이션
3. 모바일 디바이스에 설치되어 활용되는 애플리케이션
4. 웹 브라우저와 동일한 형태로 모바일상 웹 브라우저를 통해 서비스를 제공

소프트웨어 개발환경
1. 서버의 하드웨어를 사용자 관점에서 편리하고 유용하게 사용하기 위한 소프트웨어
2. 컴퓨터와 컴퓨터 간의 연결을 쉽고 안전하게 할 수 있도록 해주고 이에 대한 관리를 도와주는 소프트웨어
3. 사용자와 데이터베이스 사이에서 사용자의 요구에 따라 정보를 생성해주고 데이터베이스를 관리해주는 소프트웨어

소프트웨어 개발을 위한 전체 과정에서 발생하는 모든 항목의 변경 사항을 관리하기 위한 활동

형상 관리 절차
1. 형상 관리 대상을 정의 및 식별
2. 변경요구 관리, 변경제어, 형상 관리등 통제 지원
3. 소프트웨어 베이스라인의 무결성 평가
4. 소프트웨어 형상 및 변경관리에 대한 각종 수행결과를 기록

소프트웨어 형상 관리 도구 유형
1. 매일 개발이 완료된 파일은 약속된 위치의 공유 폴더에 복사하는 방식
2. 중앙에 버전 관리 시스템을 항식 동작시키는 방식
3. 로컬 저장소와 원격 저장소로 분리되어 분산 저장하는 방식

소프트웨어 형상 관리 도구
1. 서버와 클라로 구성, 다수 인원이 동시에 범용적인 운영체제로 접근 가능한 형상 관리 도구
2. 하나의 서버에서 소스를 쉽고 유용하게 관리할 수 있게 도와주는 도구
3. cvs 달리 소스 파일의 수정을 한 사람만으로 제한하여 다수의 사람이 파일의 수정을 동시에 할 수 없도록 파일 잠금 방식으로 형상관리
4. svn와 비슷한 중앙 통제 방식으로 대규모 프로젝트에서 빠른 속도를 냄
5. 속도에 중점을 둔 분산형 버전 관리 시스템 커밋은 로컬 푸시는 원격 저장소에 반영
6. 복수서버, 복수 클라 구조 서버가 부족할 때 필요한 서버를 하나씩 추가하여 확장성 기할 수있음
