라이브러리의 구성

라이브러리는 전역변수, 함수 등을 모아둔 ~와 디렉터리형식으로 구조화한 라이브러리인 ~로 구성된다

절차적 프로그래밍 언어 종류
1. 알고리즘 연구개발에 이용하기 위해 생성됨
2. 유닉스 운영체제 사용위해 개발, 모든 컴터 시스템에 사용할 수 있도록 개발
3. 교육용으로 개발되어 언어의 문법이 쉬움
4. 과학계산에서 필수적인 기능이 내장된 과학 기술 전문 언어

객체지향 프로그래밍 언어
1. c문법에 객체지향 개념과 일반 프로그래밍을 위한 템플릿 기능이 추가
2. 마이크로소프트에서 개발함, 불안전 코드 같은 기술을 통해 플랫폼 간 상호운용성 확보
3. 바이트 코드라는 특수한 바이너리 형태로 변환
4. 파스칼 문법에 여러 기능이 추가되어 존재

스크립트 언어
1. 동적 웹 페이지를 만들기 위해 설계
2. 인터프리터 방식의 프로그래밍 언어
3. 다양한 플랫폼에서 쓸 수 있고 라이브러리 풍부함, 유니코드 문자열을 지원
4. 객체 기반의 스크립트 프로그래밍 언어

선언형 언어 종류
1. 패턴 맞춤, 커링 등의 기능존재, 재귀함수나 대수적 자료형도 지원
2. 수학 표기법을 나타내기 위한 목적으로 생성
3. 논리식을 기반으로 객체 간의 관계에 관한 문제를 해결하기 위해 사용
4. 관계형 데이터베이스 관리 시스템의 데이터를 관리하기 위해 설계된 특수 목적의 프로그래밍 언어
