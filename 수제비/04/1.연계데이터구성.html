서로다른 두 시스템, 장치,소프트웨어를 이어주는 중계 역할을 하는 연계 시스템과 관련된 요구사항을 분석하는 과정

연계 요구사항 분석 기법 및 참고문서
1. 사용자 면담을 통해 연계 요구사항 도출
2. 시스템 운영 환경, 성능, 보안등 기준에 대한 점검을 통한 분석기법
3. 서비스 활용 목적에 따라 연계 필요 데이터 식별 및 연계 주기 등을 설문지를 통해 분석하는 기법
4. 전문가의 경험적 지식을 통한 분석 기법
5. 소속된 인원들이 자발적으로 자연스럽게 제시된 아이디어 목록을 통해 분석하는 기법

1. 공통 코드 정의서
2. 데이터 모델링 정의서, 테이블별 컬럼 속성 정의서등
3. 애플리케이션의 메뉴구성 발생패턴 등 확인
4. 하드웨어 소프트웨어 네트워크 등 연계 대상 시스템 구성도

인터넷페이스 명세서 구성요소 인최크시데
1. 인터페이스를 구분하기 위한 식별자
2. 단위 시간당 처리될 수 있는 해당 인터페이스 최대 수행 건수
3. 해당 인터페이스 1회 처리 시 소요되는 데이터의 평균 크기와 최대 크기
4. 시스템명, 업무 명, 서비스명/프로그램id 연계 방식, 담당자/연락처
5. 번호, 필드, 식별자 여부, 데이터 타입, 데이터 크기, 널 허용 등

연계 시스템의 구성
1. 연계할 데이터를 데이터 베이스와 애플리케이션으로부터 연계 테이블 또는 파일을 생성하여 송신
2. 수신한 연계 테이블 또는 파일의 데이터를 수신 시스템에서 관리하는 데이터 형식에 맞게 변환하여 데이터베이스에 저장하거나 활용 제공
3. 송신 시스템과 수신 시스템 사이에서 데이터를 송수신하고 연계 데이터의 송수신 현황을 모니터링

인터페이스 시스템의 데이터 표준
1. 인터페이스 표준 항목을 포함
2. 송수신 시스템에서 업무 처리에 필요한 데이터 포함
3. 전송 데이터의 끝을 표시하는 문자를 포함하여 종료 표시

개체의 타입과 관련 속성, 식별자 등의 정보를 개괄적으로 명세화한 정의서
