기업에서 운영되는 서로 다른 플랫폼 및 애플리케이션 간의 정보를 전달, 연계, 통합이 가능하도록 해주는 솔루션

EAI구성요소
1. 이기종 시스템 간 애플리케이션 상호 운영
2. 다양한 패키지 애플리케이션 및 기업에서 자체적으로 개발한 애플리케이션을 연결하는 EAI의 핵심 장치
3. 시스템 상호 간 데이터가 전송될 때, 데이터 포맷과 코드를 변환하는 솔루션
4. 비동기 메시지를 사용하는 다른 응용 프로그램 사이에서 데이터를 송수신하는 기술
5. 미리 정의된 기업의 비즈니스 Workflow에 따라 업무를 처리하는 기능

EAI구축 유형
1. 가장 기초적인 애플리케이션 1:1 통합방법
2. 단일한 접점의 허브 시스템을 통하여 데이터를 전송하는 중앙 집중식 방식
3. 애플리케이션 사이 미들웨어를 두어 연계하는 미들웨어 통합 방식
4. 그룹 내는 허브 앤 스포크 방식을 사용하고 그룹 간에는 메시지 버스 방식을 사용하는 통합 방식

기업에서 운영되는 서로 다른 플랫폼 및 애플리케이션들 간을 하나의 시스템(미들웨어)으로 관리 운영할 수 있도록 서비스 중심의 통합을 지향하는 아키텍처.

네트워크에 분산된 정보를 서비스 형태로 개방하여 표준화된 방식으로 공유하는 기술

웹서비스 유형
1. HTTP,HTTPS,SMTP등을 사용하여 XML 기반의 메세지를 네트워크 상태에서 교환하는 프로토콜
2. 웹 서비스명, 제공 위치, 메시지 포맷, 프로토콜 정보 등 웹 서비스에 대한 상세정보가 기술된 XML형식으로 구현되어 있는 언어
3. 웹 서비스에 대한 정보인 WSDL을 등록하고 검색하기 위한 저장소

송신 시스템과 수신 시스템을 연계 하였을 경우 데이터의 정합성과 데이터 전송 여부에 대한 테스트
